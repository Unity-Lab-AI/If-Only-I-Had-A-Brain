WHAT GEE CAUGHT (the symptoms, live box 3h window)
  • ws.bufferedAmount to the donor sawtoothed 68MB → 1.6 GB ("why is the
    buffer getting to Gigs in size?")
  • Donor row RED in the client log — median heartbeat RTT 19,185ms
  • Donor (Chrome compute.html tab) disconnected UNEXPECTEDLY every ~12min
    (+10675s, +11436s), each crash triggering full re-init + 17-matrix
    re-upload on top of the ongoing flood
  • gpuShadowDirty chronically ON (auto-resync cleared it at 2:47:22, first
    shed re-set it at 2:53:09 — re-dirtied within seconds of every clear)

ROOT CAUSE (code-audited, NOT a guess)
  Three streams share the one donor websocket:
    (a) hebbian batch frames         — guarded (TU.25.A 64MB soft-shed)
    (b) sparse binary matrix uploads — guarded (500MB await-drain)
    (c) teach-pattern JSON frames    — write_spike_slice / write_current_slice
        / clear_spike_region — **ZERO guard**: raw ws.send(), 8 region clears
        + pattern writes per teach iteration, thousands/sec during teach.
  Stream (c) is what grew the buffer to GBs. The ping then queued BEHIND that
  mass (19s RTT = red row), and the Chrome tab OOM/crashed under the receive
  backlog (12min flap cycle). gpuShadowDirty is WORKING AS DESIGNED (CPU
  authoritative, GPU shadow, F5 auto-resync arms + clears correctly) — it was
  chronically dirty because production outran the pipe, not because the flag
  or resync is broken.

THE FIX (TU.28.1-3, already in the source on this branch)
  • TU.28.1 — server/brain-server/gpu.js: `_donorPatternSendGated(json)` +
    `_donorSoftCapBytes()` gate ALL three pattern writers at the SAME soft cap
    as the hebbian shed (env DREAM_WS_SOFT_SHED_MB, default 64MB). Above cap:
    frame DROPPED (never enqueued), _gpuShadowDirty set (auto-resync heals),
    counted + rate-limited log "[Brain] TU.28.1 — teach-pattern frame SHED".
    Replica mirror gets the same per-socket gate. Gate probes unaffected
    (gpuDrainWait drains to 10MB < cap before probes fire).
  • TU.28.2 — server/brain-server/state.js: wsPressure JSON now exposes
    `patternSheds` + `mirrorSheds` so saturation is attributable per-stream
    straight from public-state.json.
  • TU.28.3 — js/brain/consolidation-engine.js: DREAM_CONSOLIDATION_MAX_MS
    default 30000 → 45000. Live passes 40/41/42 ALL DEADLINE-ABORTed at
    30.5-32.1s — tail stages (incl. Tier-3 promotion) were cut EVERY pass.
    45s finishes the pass; env knob unchanged; still guards the 153s runaway.
