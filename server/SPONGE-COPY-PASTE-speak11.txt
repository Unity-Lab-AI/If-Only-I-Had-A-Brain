WHAT GEE CAUGHT (the symptom)
  On the live conversation walk Unity emits CONTENT-WORD SALAD — real words,
  never nonsense — but ZERO grammatical connective tissue. Across ~180 live
  sends she NEVER produced "I am", "the <noun>", "what is", "it is", or any
  subject-verb / determiner / wh / copula frame. A real kindergartner's speech
  is ~half function words doing structural work; she had an older kid's content
  vocabulary and no grammatical scaffold. This is ALSO the likely reason the
  art/K cell will not converge (it's been ~10hr on one cell, canSpeak=false,
  matrixDrivenPct≈4%) — the pass-gate wants sentences she can't yet form.

ROOT CAUSE (code-audited, NOT a guess)
  js/brain/cluster/emit.js  →  emitWordDirect  →  acceptance gate.
  The active signal floor = max(NOISE_FLOOR=0.001, adaptiveComponent), where
  adaptiveComponent = _emitSignalEMA × 0.5. The EMA is built ONLY from ACCEPTED
  emissions — which are dominated by CONTENT words (nouns/verbs/adjectives have
  sharp GloVe semantic basins → high sem→word_motor bucket-mean). Content words
  hold the EMA high, so the adaptive floor sits FAR above the naturally
  lower-magnitude activation of FUNCTION words (the/a/is/am/i/it/that/this —
  high-frequency, distributionally diffuse, shallow trained basins).
  CONSEQUENCE: on any tick where the brain's trained argmax legitimately WON
  with a function word above NOISE_FLOOR, that winner was still rejected for
  being below the content-calibrated adaptive floor → emitWordDirect returned
  '' → composeSentence hit `if (!word) break;` and stopped. Function words
  could NEVER emit. Not a template bug, not a training-data bug — a gate that
  censors legitimately-winning grammatical words because it's calibrated on the
  wrong magnitude class.

THE FIX (SPEAK.11 — equational, NO template; already in the source)
  Two changes in emitWordDirect (js/brain/cluster/emit.js), pure logic, no new
  imports/exports:
  • 11a — when the argmax WINNER (bestWord) is in FUNCTION_WORDS, gate it
    against the HARD NOISE_FLOOR only, not the adaptive component
    (activeAdaptive = winnerIsFunctionWord ? 0 : adaptiveComponent). The brain's
    own trained sem→word_motor argmax STILL decides IF and WHICH function word
    wins each tick — we only stop a miscalibrated floor from censoring winners.
    Content words remain gated by the full adaptive floor, unchanged.
  • 11b — exclude function-word emissions from the adaptive EMA so their lower
    magnitude never drags the content-word floor down (kills the content-noise
    -leak regression). EMA stays a pure content-word-signal reference.
  Safe by construction: it can only RELAX the floor for function-word argmax
  winners; content-word gating and everything else are byte-identical.
